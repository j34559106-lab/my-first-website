== 나만의 커뮤니티 기획서 ==

커뮤니티 주제
- 맛집 정보 공유

디자인 스타일
- 사이트 이름: Community
- 색상 테마: 노란색
- 분위기: 친근한
- 모바일/PC 반응형 대응

특별한 기능 아이디어
- 별점 (게시물별 평점 등록, 평균 별점 자동 계산)
- 위치공유 (지도 API 연동 - 카카오맵/네이버지도/구글맵 중 택1, 또는 최소 요건으로 주소 텍스트 입력)
- 사진업로드 (최대 장수 및 용량 제한 정하기, 필수 여부 결정)

페이지별 특징

- 로그인 페이지:
    - 이메일 입력 필드
    - 비밀번호 입력 필드
    - 로그인 버튼
    - 비밀번호 찾기 링크 → 비밀번호 재설정 페이지로 이동

- 비밀번호 찾기 / 재설정 페이지:
    - 이메일 입력 필드
    - 인증코드 발송 버튼
    - 인증코드 입력 필드
    - 새 비밀번호 입력 / 확인 필드
    - 재설정 완료 버튼

- 회원가입 페이지:
    - 이메일 입력 필드
    - 이메일 인증 버튼 (인증코드 발송)
    - 인증코드 입력 필드 + 확인 버튼
    - 비밀번호 입력 필드 (최소 길이·특수문자 포함 등 정책 정하기)
    - 비밀번호 확인 입력 필드
    - 닉네임(이름)
    - 이용약관 / 개인정보처리방침 동의 체크박스
    - 회원가입 버튼

- 게시물 목록:
    - 가게 이름
    - 지역
    - 종류 (한식/일식/중식/카페 등 음식 카테고리)
    - 연령대
    - 분위기
    - 별점 (평균)
    - 대표 사진 썸네일
    - 검색창 (가게 이름 등 텍스트 검색)
    - 정렬 옵션 (최신순 / 인기순 / 별점순)
    - 목록 표시 방식 (페이지네이션 또는 무한스크롤 중 결정)

- 게시물 작성 / 수정 페이지:
    - 가게 이름
    - 지역
    - 종류
    - 연령대
    - 분위기
    - 위치(지도 첨부 또는 주소 입력)
    - 별점 입력
    - 본문내용
    - 사진 업로드
    - 등록/수정 버튼

- 게시물 상세:
    - 가게 이름
    - 지역
    - 종류
    - 연령대
    - 분위기
    - 위치(지도)
    - 영업시간 / 전화번호 / 가격대
    - 별점
    - 본문내용
    - 작성자
    - 사진
    - 좋아요 수 (로그인 사용자만 클릭, 토글로 취소 가능)
    - 스크랩 버튼
    - 신고 버튼

- 댓글 기능:
    - 대댓글
    - 댓글 좋아요
    - 베스트 댓글 상단 고정
    - 댓글 정렬(최신순/베스트순)
    - 댓글 신고

- 마이페이지:
    - 프로필(닉네임, 프로필 사진) 수정
    - 비밀번호 변경
    - 내가 쓴 게시물 목록
    - 내가 쓴 댓글 목록
    - 스크랩한 게시물 목록
    - 알림 목록 (댓글 알림, 좋아요 알림 등)

- 관리자 페이지:
    - 신고된 게시물/댓글 처리
    - 회원 관리(정지/차단)
    - 게시물 관리(삭제/숨김)

- 기타 페이지:
    - 이용약관 페이지
    - 개인정보처리방침 페이지

참고 사이트
- 비슷한 느낌의 웹사이트가 있다면: 네이버 카페

향후 결정 필요 사항 (개발 시작 전 확정 권장)
- 위치공유 방식: 지도 API 사용 여부 및 어떤 API를 쓸지
- 소셜 로그인(카카오/구글) 도입 여부
- MVP(최소 기능 제품) 범위: 신고/알림/관리자 페이지는 초기 버전에서 제외하고 추후 추가할지 여부
